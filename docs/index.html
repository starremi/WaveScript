Completed contribution

Person A: Language and Interpreter

Person A work covers the path from WaveScript source code to program execution and sound event generation. The interpreter produces the event list that the audio, visualization, and sheet music backends use.

  • TextX grammar in grammar/wavescript.tx
  • Parser support for .wave and .ws style programs
  • Interpreter logic for let, if, else, for, and print
  • Sound event generation with pitch, frequency, start time, and duration

Project roles

How the Team Split the Work

Person A: Language and Interpreter

Jania's part takes WaveScript source code through the TextX parser, interpreter, and event generator.

  • Defines the WaveScript grammar
  • Parses programs into a structured model
  • Runs variables, conditionals, loops, functions, and print behavior
  • Creates sound events for the backend tools

Person B: Audio, Visualization, and Sheet Music

Greta's part starts with the shared event list and turns it into the final creative outputs.

  • Synthesizes audio files from events
  • Generates waveform images and optional spectrograms
  • Exports printable LilyPond sheet music
  • Supports commands like show waveform audio and export score "output.ly"

Language design

Core WaveScript Ideas

Musical Syntax

Statements combine notes, durations, arrows, and commands.

C4 q -> print "Hello"

Dual Meaning

The same program can print output and create a timed musical phrase.

musical syntax -> sound events

Shared Event Format

Each generated event can be passed to audio and visualization tools.

{ pitch, frequency, start, duration }

Sample programs

Example WaveScript Code

Hello World

tempo 120
C4 q -> print
melody "Hello"
G4 h -> end

FizzBuzz

tempo 140

D4 q -> for i in 1..15
    E4 e -> if i % 15 == 0
        C5 s -> print "FizzBuzz"
    F4 e -> else if i % 3 == 0
        A4 s -> print "Fizz"
    G4 e -> else if i % 5 == 0
        B4 s -> print "Buzz"
    C4 q -> else
        C4 q -> print i

G4 w -> end

Generated output

From Events to Sound

WaveScript generates timed sound events first. Backends can then synthesize audio, render waveform images, and export LilyPond sheet music.
